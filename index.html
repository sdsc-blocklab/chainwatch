ChainWatch

Blockchain Network Monitoring and Analysis

SPARK AI - BlockLAB
San Diego Supercomputer Center
Paper Code Poster

Abstract

The objective of this project is to deploy a blockchain network on the Comet supercomputer at SDSC, with the goal of analyzing blockchain network performance and security in a series of simulated, real world scenarios. In creating a blockchain network in the HPC environment we will be able to simulate a large distributed network enabling stress testing and the collection of data relating to real-world scenarios. Our plan is to then utilize the collected data to investigate automated, proactive decision rules with the purpose of analyzing strategies to improve network security and performance.

Motivation

The SDSC Blockchain and HPC team has been experimenting with high performance blockchain networks in HPC environments. Specifically, the team has worked with VMware’s blockchain service and open source protocol (SBFT) to run a simulated real-world blockchain network in a contained computing environment (1). Using Dell Boomi commands we have populated this network with over 500 synthetic HPC jobs. A technical report summarizing this work has been completed, outlining the research and proposes future research topics and use cases.

One prominent use case that has been theoretically explored was enhancing blockchain security through the analysis of large scale blockchain networks, simulated on HPC. Specifically, using pre-emptive, off-chain information such as, network monitoring and analysis to proactively detect attacks. Currently, blockchain cybersecurity attacks are predominantly handled reactively, often resulting in destructive techniques such as hard forks. As the importance of on-chain data intensifies and the value stored in smart contracts and digital wallets increases, the protection of blockchain networks becomes of greater importance. This network and blockchain activity monitoring research would be designed to create a labeled dataset of information/traffic on an HPC blockchain network. The dataset(s) created by this project could be used to explore the collaboration between AI/ML network monitoring and blockchain network security.

ChainWatch

Our team has deployed a private Ethereum blockchain network which is being hosted on computing instances at the San Diego Supercomputer Center (2). We chose to use Ethereum due to its widespread adoption, thus more information on it and also as there are many network monitoring resources. Creating a private network means that only nodes we approve will be allowed to join the network, as we are starting all the nodes ourselves approving which nodes may join will be very trivial.

We are spinning up and connecting Ethereum nodes to our private Ethereum blockchain using Docker and Go Ethereum (geth) (3). Geth is one of the three original implementations of the Ethereum protocol and allows rapid creation of Ethereum nodes to interact with an ethereum blockchain. Docker is a tool to run an application in an isolated environment (4). We are using Docker to isolate and scale deployment of nodes to simulate a real world environment. There are many tools for monitoring the network information of docker containers that make Docker an optimal tool for the ChainWatch project.

Eth-netstats, along with Docker stats CLI are being used to monitor the activity of the nodes and collect a dataset of network activity labeled with specific events and types of interactions (5). Eth-netstats uses the nodes Json-rpc server to interact and monitor individual nodes. Eth-netstats is able to monitor and record hash rates, last block mined, average block time and other data points. See figure 1 for Eth-netstats GUI. The Docker stats CLI command is used to stream a container's runtime metrics including network IO and memory usage.

Architecture

Blockchain Network Simulation and Data Collection

  • Ethereum Network Running on SDSC Computing Instance using Docker
  • Deploying Simple ERC-20 Smart Contract to network
  • Automated transactions among accounts on network
  • Can simulate different levels of traffic
  • Network setup is completely automated
  • Collecting transaction data, docker data (hardware data) and node data (blockchain data)
    • Collection was split up into two distinct categories: heavy traffic (1000 tx/minute), and light traffic (1 tx/minute)


Analysis

  • Merge light and heavy transaction logs and information about the nodes and network
    • Remove identifying information (timestamps, blockHash, transactionHash, etc.)
  • Visualize node information in context with transaction types
    • i.e. memory usage, net input/output, CPU usage
  • Build machine learning model to classify between heavy and light transaction based on node usages
    • Supervised classification (Random Forest, Logistic Regression, Nearest Neighbors, etc.)
    • Unsupervised — Clustering (KMeans, hierarchical: agglomerative/divisive)

Summary

Improving the security and performance of blockchain networks is of increasing importance as blockchain technology advances and widens in adoption. Chainwatch uses the simulation of real world blockchain networks which will allow us to collect data in association with different events which can happen on a blockchain network. Aggregating pre-validated off-chain data associated with specific blockchain network activity, will offer insight and solutions into ways in which blockchain networks can improve their security and performance.
